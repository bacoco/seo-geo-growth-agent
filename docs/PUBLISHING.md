# Publishing to GitHub

This folder is already a local Git repository once generated by the packaging script.

## Option A: GitHub CLI

Create an empty GitHub repository and push:

```bash
cd seo-geo-growth-agent
gh repo create seo-geo-growth-agent --public --source=. --remote=origin --push
```

For a private repo:

```bash
gh repo create seo-geo-growth-agent --private --source=. --remote=origin --push
```

## Option B: GitHub website + git remote

Create an empty repository on GitHub named `seo-geo-growth-agent`, without adding a README, license, or `.gitignore`, then run:

```bash
cd seo-geo-growth-agent
git remote add origin git@github.com:YOUR_USER_OR_ORG/seo-geo-growth-agent.git
git branch -M main
git push -u origin main
```

HTTPS alternative:

```bash
git remote add origin https://github.com/YOUR_USER_OR_ORG/seo-geo-growth-agent.git
git branch -M main
git push -u origin main
```

## Option C: API script

The helper script uses environment variables and never asks you to paste a token into the repository files.

```bash
export GITHUB_TOKEN='YOUR_TOKEN'
export GITHUB_OWNER='YOUR_USER_OR_ORG'
export GITHUB_REPO='seo-geo-growth-agent'
export GITHUB_PRIVATE='false'
./scripts/create_github_repo.sh
```

Then push:

```bash
git remote add origin git@github.com:$GITHUB_OWNER/$GITHUB_REPO.git
git push -u origin main
```

## Release ZIP

To build a clean release archive without `.git`:

```bash
./scripts/package_release.sh
```

The output will be written to `dist/`.
