# 03 — Content Production SOP

## Page creation workflow

1. Confirm target query and search intent.
2. Assign one primary keyword and one primary URL.
3. Choose page type: comparison, pricing, category, use-case, tutorial, template, glossary, report.
4. Draft an answer-first outline.
5. Add evidence: numbers, experience, screenshots, product details, customer examples, or public sources.
6. Add structured table: key facts, comparison, steps, criteria, pricing notes, pros/cons.
7. Add internal links to and from related pages.
8. Add CTA blocks appropriate to funnel stage.
9. Add schema that matches visible content.
10. Publish, update sitemap, request indexing or push IndexNow when appropriate.
11. Track in the keyword map and report after the measurement window.

## Content quality rules

- Use real product facts and first-hand experience.
- Avoid generic claims like “revolutionary,” “seamless,” or “best-in-class” unless supported.
- Replace vague statements with numbers, constraints, dates, and examples.
- Use tables where they help AI engines extract answers.
- Keep FAQs visible on the page; do not add schema for hidden or absent content.
- Put the current year only when freshness is genuinely relevant.
- Make comparison pages fair. Include tradeoffs and who should choose the competitor.

## CTA patterns

### Strong BOFU CTA

```markdown
> Ready to evaluate [PRODUCT]? Start with [low-friction action] and see whether it fits [USE_CASE].
>
> [CTA_BUTTON]([CTA_URL]?utm_source=organic&utm_medium=seo&utm_campaign=[slug]&utm_content=cta-bofu)
```

### Medium MOFU CTA

```markdown
> Comparing [CATEGORY] options? Use [PRODUCT] to [specific outcome] without [common pain].
>
> [CTA_BUTTON]([CTA_URL]?utm_source=organic&utm_medium=seo&utm_campaign=[slug]&utm_content=cta-mofu)
```

### Soft TOFU CTA

```markdown
> Want to turn this guide into an actual workflow? [PRODUCT] helps [AUDIENCE] move from theory to implementation.
>
> [CTA_BUTTON]([CTA_URL]?utm_source=organic&utm_medium=seo&utm_campaign=[slug]&utm_content=cta-tofu)
```
